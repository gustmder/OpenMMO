# Devlog

Development notes for notable project changes, asset additions, and design decisions.

## Entries

| Date | Topics | Notes |
|------|--------|-------|
| [2026-07-17](2026-07-17.md) | Google Sign-In | Replaced the password login with Google Sign-In: the login screen shows a single Google button and the server verifies the ID token. The old password path was retired for security; new accounts get a random `player_xxxxxx` name and no email or profile name is stored. |
| [2026-07-16](2026-07-16.md) | Off-Hand Shields, Solid Furniture Blocks Movement | Added Wooden and Raven off-hand shields, each with a Guard stat that now feeds into combat and shows on the character sheet. Solid furniture now seals the tiles it sits on so players and NPCs walk around it instead of through it; the collision logic is shared Rust, so the browser, bots, and server all agree on which tiles are blocked. |
| [2026-07-14](2026-07-14.md) | Dressing Up Rica's Shop | Hung a procedural "Rica's General Store" sign, decorated the shop interior with item models placed as furniture, and updated Rica's stock to sell Healing Potions and Scrolls of Return. |
| [2026-07-13](2026-07-13.md) | Horizontal World Wrapping | Completed the partial X-wrap implementation so the east and west edges now connect without fallback terrain or a stuck map marker. |
| [2026-07-12](2026-07-12.md) | Water Shader Rework | Rewrote the sea and river shaders, unified them so sea-river boundaries blend seamlessly, improved ocean wave motion, and added splash effects around river obstacles such as rocks and bridge posts. |
| [2026-06-30](2026-06-30.md) | Quickslot Bar | Added a 10-slot quickslot bar across the bottom of the screen. Drag a bag item onto a slot to bind it (showing the icon plus remaining bag count); double-click or press the slot's number key (1–9, then 0) to use it — equippables equip, consumables drink — and right-click clears it. Bindings are keyed by item *definition* (not instance) so they survive stacks being consumed/recreated, and persist per character in localStorage. Also reorganized the bottom HUD: chat, quickslots, and menu buttons now live in one flex row (chat shrinks on the left, quickslots + menu hug the right and wrap to two rows when narrow) instead of three independently-positioned elements, and a shared drag hit-test keeps the drop highlight aligned with the actual drop target. |
| [2026-06-28](2026-06-28.md) | Healing Potion | Added the first drinkable consumable: double-click a Healing Potion in the bag and the server rolls 6d4 HP (NetHack dice) capped at max — a full-HP or downed character keeps it instead of wasting it. Routed via a new `UseItem` message; one is consumed per drink and the health change broadcasts to nearby players. Also folded the `damageDice`/`healDice` item columns into a single `category` + `dice` pair, where `category` (`weapon`/`healing_potion`/`armor`/`accessory`/`currency`) decides how `dice` is interpreted (weapon → damage, healing_potion → heal). |
| [2026-06-27](2026-06-27.md) | Corridor Doors | Corridor mouths into rooms now get a click-to-open wooden double door (same leaves and stone arch as the surface entrance) instead of a bare gap. The leaves and arch sit flush on the wall plane, and corridor wall runs are trimmed where they meet a room wall so the corner stops z-fighting. |
| [2026-06-25](2026-06-25.md) | Distinct Corridor Walls | Dungeon corridors now use a separate wall texture from rooms. Rooms keep the medieval stone blocks; corridors (carved cells in no room or shaft — the same half-open room test the Rust generator uses) get a rock wall instead, with wall runs breaking wherever the room↔corridor texture flips. The corridor texture is [Rock Wall 10](https://polyhaven.com/a/rock_wall_10) from Poly Haven, picked after trialing a batch of Poly Haven stone/plaster walls in-engine (runner-ups kept under `textures/dungeon/`), UV scale `0.8` so the stones read larger. Reuses the shared housing material pipeline and per-run ghost-fade, so no new rendering machinery. |
| [2026-06-20](2026-06-20.md) | Dungeon Click Pathing | Fixed a dungeon entrance-stair click bug where clicking the red-marked floor area in the room routed the player back outside instead of moving there. The culprit was the hidden surface entrance building: Three.js raycasts can still hit invisible objects, so its walls/roof intercepted underground floor clicks. Ground raycasts now target only the active dungeon floor group, and floor resolution treats the entrance shaft as dungeon view. |
| [2026-06-18](2026-06-18.md) | Breakable Dungeon Props, Interactive Chests & Coin Pickups | Barrels and crates are now breakable: click one, the player walks up and swings once, and the whole-model GLB swaps to its broken-debris variant at the contact frame (chests stay intact). Server-authoritative — `DungeonRuntime.broken_props` validates floor/proximity/kind, broadcasts `DungeonPropBroken` via floor-gated AOI, and sends a `DungeonPropsState` snapshot on floor entry. The smashed cell becomes walkable via a `dungeon_apply_broken_props` cache rebuild. Also fixed a wasm-vs-native `usize` RNG desync that made client and server disagree on prop cells. Dungeon chests are now interactive end to end: clicking one walks the player up and swings the lid open (`chest_animated.glb`'s `ChestOpen` clip, ~105°), server-validated and synced via `DungeonPropOpened` with an `opened` list in the floor-entry snapshot. Opening spawns a real `coin_pile` ground item in front of the chest (riding the generic ground-item pipeline, playing `coin_pile_spill.glb` to a settled heap); picking it up credits a random 1–10 copper straight to the wallet instead of a bag slot, and the pile never raises to the hand during the pickup gesture. |
| [2026-06-17](2026-06-17.md) | Stair Passability | Blocked passability flags so current-floor monsters can't enter the high side of the stairs (they were marching up the shaft and clipping through the floor above). The new `wall_shaft` closure seals the far landing's room-facing run-end while leaving the steps open for descending players. The red band in the screenshot is a passability debug overlay, not a gameplay element. |
| [2026-06-15](2026-06-15.md) | Torch Shadow Artifacts | Fixed dungeon torch-light shadow glitches. Key detail: three.js records the shadow map from the **backface** by default (`FrontSide` materials derive `shadowSide = BackSide`), which self-biases against acne but peter-pans contact shadows — fixed with a 0.02m `SHADOW_CONTACT_LIFT` plus a larger torch shadow map and smaller bias. |
| [2026-06-14](2026-06-14.md) | Procedural Dungeons | Added seed-deterministic, NetHack-style maze dungeons (fixed per entrance, up to 20 floors). Surface entrances reuse the housing procedural geometry as a stone building, with terrain holed out to expose the stairs. The final floor holds a boss with loot. |
| [2026-06-11](2026-06-11.md) | Economy Phase 2: LLM Haggling | Rica's LLM can now grant per-player price deals via the new `offer_deal` action; the server clamps offers to a CHA-derived price band, enforces daily budgets and cooldowns, logs every decision, and the trade window shows haggled prices. |
| [2026-06-10](2026-06-10.md) | Goblin, Player Control FSM, Economy Phase 1 | Added the goblin monster, converted player control to an explicit FSM, and implemented single-currency gold plus fixed-price merchant trading with Rica (basePrice data, trade protocol, server validation, trade window UI). |
| [2026-06-08](2026-06-08.md) | Kobold, Small Sword | Added the kobold monster, documented the 2D -> 3D asset workflow, adjusted the kobold scale, and added the small sword weapon. |
